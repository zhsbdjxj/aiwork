┌─────────────────────────────────────────────┐
│  PPT P5：我们的解题思路 —— 从上往下定义标准     │
├─────────────────────────────────────────────┤
│                                              │
│  传统做法：从下往上拼数据 → 每接新标段=重新适配 │
│                                              │
│  我们的做法：                                  │
│  制定统一标准 → 下级按标准接入 → 数据天然统一   │
│  ✓ 新标段接入 = 即插即用                       │
│  ✓ 天生标准一致，无需后期拼凑                   │
│                                              │
│  一句话：别人在拼积木，我们在画图纸。           │
│  ★ 更关键的：标准在青海跑通后，                 │
│    复制到下一个省时，80%的工作已经完成。         │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐

│                                              │
│  【编制过程：不是闭门造车】                     │
│  ┌─────────────────────────────────────┐    │
│  │  ① 横向调研：参考甘肃、新疆、四川、西藏    │    │
│  │     等多个省份项目管理平台的数据规范      │    │
│  │  ② 内部论证：结合公司在青海标段级业务     │    │
│  │     系统积累的数据实践经验              │    │
│  │  ③ 客户对齐：与青海公路局多轮沟通确认——  │    │
│  │    "业主需要的不是技术规范，是管得住的    │    │
│  │     数据口径"                         │    │
│  │  ④ 终版下发：形成《青海省公路工程         │    │
│  │     智慧工地数据接入标准指南》V1.0        │    │
│  └─────────────────────────────────────┘    │
│                                              │
│  【客户下发标准，想达到什么效果？】              │
│  ┌─────────────────────────────────────┐    │
│  │ ✓ 统一口径：全省所有标段按同一套规范上报   │    │
│  │    "不管谁建的系统，说同一种数据语言"    │    │
│  │ ✓ 横向可比：质量/进度/人员数据可跨标段   │    │
│  │    对比——"谁好谁差，数据说了算"        │    │
│  │ ✓ 掌握主动：业主成为数据标准的制定者，   │    │
│  │    而非被动接收者——"用什么格式，局里定" │    │
│  │ ✓ 为未来铺路：标准一旦建立，新增项目     │    │

│                                              │

│  ┌─────────────────────────────────────┐    │
│  │ ✓ 锁定入口：标准由我们编制、由业主下发，  │    │

│  │    先定义规则者，掌握生态主导权        │    │

│  │ ✓ 建立壁垒：竞品要进入，必须适配       │    │

  关键词：30秒、全局、实时、自动预警、率先垂范

【角色B：项目办/总包管理层】
  关注点：跨标段对比、进度质量管控、汇报效率
  痛点话术：

  价值话术：
  "平台自动汇聚各标段数据，横向对比一键生成。视频AI轮巡帮您自动盯着工地安全。眼镜让一线巡检效率翻倍。"
  关键词：对比、管控、自动、效率

【角色C：标段现场负责人】
  关注点：施工安全、巡检效率、资料留痕、检查应对
  痛点话术：

  关键词：省事、省心、自动、留痕、应付检查

----------------------------------------------------------
  4.2 按业务场景分类话术
----------------------------------------------------------

【场景一：隧道安全】
  "隧道内视频监控+AI轮巡，自动识别违规进入、异常情况。巡检员戴AR眼镜进入，第一视角实时回传地面。"
  价值：隧道安全不是'出了事再处理'，而是'冒头就掐灭'。

【场景二：梁场管理】
  "平台接入梁场质量检测数据，关键指标自动对比标准值，异常自动标红。眼镜扫构件→显示批次、检测状态、责任人。"
  价值：梁场最怕'事后知道'，我们让质量在每一步都透明。

【场景三：人员管理（发卡机/闸机）】
  "产业工人平台劳务数据→眼镜人脸识别→资质/考勤/培训一键浮现。无证上岗实时告警，黑名单自动拦截。"
  价值：管人最怕'没想到'，眼镜一扫全知道。

【场景四：日常巡检】
  "眼镜逐项引导任务→自动拍照+定位+时间戳→语音标注→自动上传平台→推送给责任人。"
  价值：巡检最怕'走过场'，系统全程留痕谁也糊弄不了。

  4.3 客户常见异议与标准应答（问题库）
----------------------------------------------------------

【异议1】"我们已经有监控系统了，为什么还要你们的平台？"
  应答："已有的摄像头我们直接接入，不用换。AI轮巡帮您自动看——给您现有的摄像头装上'脑子'。"
  落脚点：利旧降本 + AI替代人工

【异议2】"这东西听起来挺复杂，我们的人能不能用起来？"
  应答："领导看大屏和看网页一样简单。一线用眼镜语音控制，说'开始巡检'就行。驻场培训手把手教。"
  落脚点：操作极简 + 驻场陪跑

【异议3】"数据都集中在平台上，安全怎么保证？"
  应答："私有化部署，数据物理上不出省。权限分级，操作全程留痕可审计。"
  落脚点：数据不出省 + 全程可审计

【异议4】"竞品也在做类似的东西，你们有什么不一样？"
  应答："我们是'从上往下'定义标准，不是从下往上拼数据。平台+眼镜形成闭环，青海已经有落地试点。"
  落脚点：顶层设计 + 软硬闭环 + 已有落地

【异议5】"预算有限，能不能便宜点？"
  应答："一起算账：巡查频次降多少→省多少差旅？AI替代几个盯屏人→省多少人力？一次事故避免→省多少返工？分阶段交付，按阶段付款。"
  落脚点：ROI算账 + 分阶段降风险

【异议6】"为什么要在青海做标准化？只是为了复制到别的省？"
  应答："标准化首先为青海自己省钱——省内几十个标段，每新增一个接入成本越来越低。向外复制是水到渠成，不是额外成本。青海作为首发省建立的是行业标杆地位。"
  落脚点：标准化降本 + 标杆地位

【异议7】"AR眼镜在工地上扛得住吗？工人学得会吗？"
  应答："工业三防不是消费级玩具。语音控制比手机App还简单。先小批量试点，扛得住再推广。数据实时上传，眼镜坏了数据不丢。"
  落脚点：工业级设备 + 低门槛操作 + 数据安全

  应答核心：不漏报优于不误报 + 人工确认2秒搞定 + 告警分级推送
  价值落脚：AI省99%盯屏时间，剩下1%点一下的事

  价值落脚：工业级不是消费级，试了就知道

  价值落脚：交钥匙+陪跑，长期合作不是一锤子买卖
