## 车辆调度系统

### 架构模式

所有页面共享同一布局骨架，**各文件独立复制**（未抽取公共模板）：

- **侧边栏**（220px 宽，`#1a2639` 深色背景）：导航菜单，class `.sidebar` / `.menu-item`，active 状态带左侧蓝色指示条 `#4f9cf7`
- **顶栏**（`.topbar`）：页面标题 + 用户信息/角色标签
- **主内容区**（`.main-container`）：浅灰背景 `#f4f7fb`

### 页面清单与职责

| 文件 | 功能 |
|---|---|
| `index.html` | 首页/入口，包含侧边栏导航和 iframe 页面切换逻辑 |
| `工作台.html` | 仪表盘，统计卡片 + 图表 + 快捷入口 |
| `调度中心.html` | 核心页面，总览 + 列表 + 日历 + 月度四视图 |
| `订单中心.html` | 订单 CRUD，筛选 + 表格 + 详情抽屉 |
| `档案中心.html` | 车辆/司机档案管理，信息卡片 + 列表 |
| `审核管理.html` | 审批流程，待审列表 + 操作按钮 |
| `结算中心.html` | 费用结算，统计 + 对账明细 |
| `系统设置.html` | 角色/权限/参数配置 |
| `车辆调度-用户端.html` | C 端用户下单页面 |
| `车辆调度-司机端.html` | 司机接单/执行页面 |

### 设计约定

- 色系：浅色背景 `#f4f7fb`，侧边栏深色 `#1a2639`，主蓝色 `#4f9cf7`
- 字体：`'Inter', -apple-system, ...`，无衬线
- 表格/列表/卡片为主的信息展示模式
- 需求文档：`丽江车辆调度管理系统-PRD.doc`，修改页面逻辑前应先参考 PRD

### 关键注意事项

- 侧边栏和顶栏的 HTML/CSS 在**每个文件中完整重复**，修改导航结构需同步所有文件
- `index.html` 可能包含 iframe 方式加载子页面，修改子页面名称时需检查一致性
- 各页面间通过 URL 参数或 localStorage 传递上下文（如订单 ID、车辆 ID）

## AR巡检眼镜

### 架构

单文件移动端原型 `APP端.html`，模拟 390×780px 手机屏幕。采用 CSS 自定义属性（Design Tokens）体系，所有色值/间距/圆角集中在 `:root` 中。

### 页面结构

- **4 个 Tab 页**（底部导航切换）：工作台 / 识别 / 协同 / 我的
- **二级详情页**（`.secondary-page`）：绝对定位覆盖层，用于展示任务详情、识别记录详情、蓝牙扫描等
- **悬浮锁屏按钮**（`.float-lock`）：右下角浮动

### 设计令牌（`:root` 变量）

修改主题只需调整变量值，所有组件自动跟随：

- 主色：`--color-primary: #2563EB`
- 背景层级：`--bg-app`（白）/ `--bg-surface`（浅灰）/ `--bg-elevated`（中灰）
- 文字层级：`--text-primary` / `--text-secondary` / `--text-tertiary`
- 语义色：`--color-success` / `--color-warning` / `--color-danger`
- 阴影/边框/间距均有对应变量

### JS 架构约定

- IIFE 包裹，不污染全局
- `DOM` 对象集中缓存所有 DOM 引用
- `state` 对象集中管理状态
- 事件委托优先于逐个绑定（`e.target.closest()` 模式）
- `safeParseJSON()` 包裹所有 `JSON.parse`
- 动态生成的 HTML 使用模板字面量，样式通过 CSS 类名控制（避免内联 `style`）

### 编码偏好（已在之前对话中确立）

- **亮色主题**优先（白底/浅灰背景）
- SVG 图标优先于 Emoji（专业工业风格）
- 避免霓虹色渐变、大面积玻璃拟态、脉冲发光等"AI 生成痕迹"
- 组件设计克制，圆角适度（6~20px），阴影轻量

# 产品低保真原型绘制规范（永久记忆）

当需要绘制或描述低保真原型时，自动应用以下规则，无需每次重复说明。

## 1. 绘制标准（ASCII 字符）
- 边框：`+---+` 与 `|   |`，标题用 `== 标题 ==` 包裹。
- 按钮：`[ 确定 ]`，输入框：`( 请输入 )`，下拉：`{ 请选择 }`。
- 列表：`• 项目` 或 `> 项目`，图标用 `[✕]`、`[✓]` 表示状态。
- 整体宽度：**60 ~ 80 字符**，保持终端和 Markdown 清晰。

## 2. 输出必须包含三部分
- **布局线框图**：明确展示导航、主内容、侧边栏、底部操作区的相对位置。
- **交互说明**：标注关键点击、输入、跳转逻辑（如“点击 [新增] → 弹出抽屉表单”）。
- **状态标注**：至少标注**默认态**、**空状态**、**加载态**和**错误态**的占位或提示文案。

## 3. 设计原则（强制）
- **浏览动线**：Web 端采用 F 型，移动端采用 Z 型，主按钮（Primary CTA）置于视觉中心或右下角（拇指热区）。
- **网格对齐**：间距用空格均匀体现，暗示 8px 网格系统。
- **一致性**：同类组件（如按钮、输入框）尺寸和样式必须统一。

## 4. 附加信息（每页必附）
- **页面目标**：用户进入此页面的核心任务是什么。
- **异常流程**：网络断开、数据为空、提交失败时的处理方式（文案或跳转）。

## 5. 默认输出规格
- 若无特殊指定，默认输出 **Web 端（1440×900）** 布局。
- 若有移动端需求，请明确说明，我将调整为 **375×812** 比例。
